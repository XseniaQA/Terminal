Client_Server
1) Прочиать про клиент-серверную архитектуру

Клиент-серверная архитектура - это вычислительная или сетевая архитектура, в которой
нагрузка распределена между поставщиком услуг - сервером и заказчиком услуг - клиентом. 
Другими словами - это ПО. Обычно данные программы (клиент и сервер) расположены на разных машинах и взаимодействуют друг с другом через вычислительную сеть с помощью сетевых протоколов, но могут располагаться и на одной машине.

2) Что ткое HTTP и HTTPS 

Hiper Text Transfer Protocol - широко распространенный протокол передачи даных, является протоколом прикладного (верхнего, 7-го) уровня. Актуальная версия 1.1. Используется в клиент-серверной архитектуре, с помощью данного протокола осуществляется обмен данными между пользовательским приложением и веб-сервером. Так же HTTP используют как протокол передачи информации для других протоколов прикладного уровня, например, SOAP.
HTTPS - является расширением протокола HTTP, это секъюрный протокол, который обеспечивает шифрование данных, для их защиты.

3) HTTP методы

Для того чтобы сервер понял, что мы от него хотим используют типы HTTP-запроса или методы HTTP. Методы могут быть безопасными, идемпотентными и кэшируемыми. 

Выделяют 4 основных метода запросов: 
GET - метод запрашивает содержимое конкретного ресурса, получает данные и никак не должен их изменять.
POST - создаёт новый ресурс из переданных данных в запросе.
PUT - изменяет содержимое запроса по указанному URI
DELETE - удаляет конкретный ресурс по указанному URI.

HEAD - запрос ресурса так же, как и метод GET, но без ответа тела.
CONNECT - установить "туннель" к серверу, оборудованному по ресурсу.
OPTIONS - используется для описания параметров соединения с ресурсом.
TRACE - выполнение задачи возвращаемого тестового сообщения.
PATCH - используется для частичного изменения ресурса.


4) HTTP статус коды сервера

Код сосотояния HTTP (статус код) - это часть первой строки ответа сервера при запросах по протоколу HTTP. Сосотоит из трехразрядного десятичного числа, первая цифра которого указывает на класс сотояния. За кодом обычно следует строка с поясннением причины именно такого ответа.
Существует 5 классов статус кодов:
1. 100-199 - информационные
2. 200-299 - успешные
3. 300-399 - перенаправления
4. 400-499 - ошибки на стороне клиента
5. 500-599 - ошибки на стороне сервера
 


5) Что такое ядро браузера 

Это программа, преобразующя содержимое веб-страниц и информацию о форматировнии в интерактивное изображение форматированного содержимого на экране. Обычно его используют в веб-браузерах, почтовых клиентах и других программах, нуждающихся в отображении и редактировании содержимого веб-страниц.

7) Что такое API
API (Application Programming Interface или интерфейс программирования приложений) 
— это совокупность инструментов и функций в виде интерфейса для создания новых приложений, 
благодаря которому одна программа будет взаимодействовать с другой

8) Что такое ендпоинты

ENDPOINT - это само обращение к маршруту отдельным HTTP методом. 
Эндпоинты выполняют конкретную задачу, принимают параметры и возвращают данные клиенту.

9) URL (URI, URL, URN)

URL (Uniform Resource Locator) - это уникальный адрес сайта в сети, который определяет его местонахождение в сети интернет.

URN (Uniform Resource Name) - это неизменяемая последовательность символов, определяющая только имя некоторого ресурса. 
Это имя определяет только название ресурса, но не говорит как к нему подключаться.

URI (Uniform Resource Indentifer) - это строка символов, которая используется для идентификации ресурса по его адресу или по его имени, 
либо по тому и другому. Соотсетственно может содержать с себе URL и  URN.

10) Идемпотентные HTTP методы

Идемпотентным метод считается, если повторный идентичный запрос, сделанный один или несколько раз подряд, имеет один и тот же эффект.
Идемпотентные методы : GET, HEAD, PUT, DELETE. Метод POST не будет считаться индепотентным, т.к. если его вызвать несколько раз, 
то он добавит несколько строк. При этом DELETE считается индепотентным, даже при том, что возвращаемый статус код отличается, 
т.к. при первом удалении код будет 200 (успешное удаление) при всех последующих запросах код будет 404 (не найден, т.к. страница удалена).

11) Безопасные HTTP методы

Безопасные методы проводят операции "только чтение": GET, HEAD, OPTIONS. Все безопасные методы являются идемпотентными. 

Существуют также кэшируемые ментоды.
Кэшируемые ответы  - это HTTP-ответы, которые могут быть закэшированы для дальнейшего восстановления и использования позже, 
тем самым снижая число запросов к серверу. Не все HTTP-ответы могут быть закэшированы. GET, HEAD - кэшируемые. 
POST, PATCH могут быть закэшированы, если установлен заголовок Content-location (en-US). 
PUT, DELETE - некэшируемые и результат их выполнения не кэшируется.

12) Идентификация, Аутентификация, Авторизация

Идентификация - это процесс распознавания пользователя по его идентификатору (например - имени)

Аутентификация - это процедура проверки подлинности, доказательство, что пользователь именно тот, за кого себя выдаёт.

Авторизация - предоставлениеопределенных прав пользователю.

13) Что такое IP

IP-адрес — уникальный сетевой адрес узла в компьютерной сети, построенной на основе стека протоколов TCP/IP

14) Что такое октеты в DNS

DNS (Domain Name Sistem) - система доменных имён. Смысл её в том, что каждому цифровому IP-адресу соответствует понятное буквенное имя - домен.
Т.е. когда мы в браузере вводим название сайта сервер DNS преобразует его в IP-адрес. 
Ответ от сервера приходит так же в виде IP-адреса и происходит обратная операция - преобразование IP-адреса в понятное буквенное имя
все тем же DNS сервером.

IP-адрес представляет собой число размером 32 бита (или 4 байта), которое может быть записано в любой системе счисления (тут речь про адрес протокола IP version 4, в IPv6 он имеет размер 128 бит). Адрес делится на 4 октета, по 8 бит каждый, которые могут иметь значение от 0 (00000000) до 255 (11111111).

15) Что такое порт, сколько портов у Linux сервера

Порт - это идентификатор для определения получателя, т.е. Пакет идет до IP компьютера.Дальше, чтобы компьютер понял, какое приложение будет обрабатывать информацию (видео, текст и т.д.) и необходим порт. Программы "договариваются", что данные друг для друга пересылаются с определенной пометкой - номер порта.

У Linux сервера есть 3 разных типа сетевых портов:
0-1023 - зарезервированы для операционной системы и основных сервисов.
1024-49151 - зарегистрированные порты, которые могут использоваться приложениями, службами, пользователями.
49152-65535 - динамические или частные порты.


16) Уровни OSI (Open Systems Interconnection model)

Сосотоит из семи уровней, которые расположены от большего к меньшему.
7) Прикладной уровень (applicition layer)
6) Уровень представления (presentation layer)
5) Сеансовый уровень (session layer) 
4) Транспортный уровень (transport layer)
3) Сетевой уровень (network layer)
2) Канальный уровень (lata link layer)
1) Физический уровень (physical layer)

На 7 уровне информация представляется в виде данных, на 1 уровне в виде бит. 
Процесс, когда информация отправляется и переходит из данных в биты, 
называется инкапсуляцией. 
Обратный процесс - декапсуляцией.

17) Хедеры http запросов

Заголовки HTTP являются основной частью НТТР-запросов и ответов и несут информацию о браузере клиента, запрошенной странице, сервере.
Наиболее распространенный хедеры НТТР-запросов:

HOST
НТТР-запрос отправляется на определенные IP-адреса. 
Но так как большинство серверов способны размещать несколько сайтов под одним IP, они должны знать, какое доменное имя ищет браузер.
Это в основном HOST включая домен и поддомен (Host: net.pampam.com)

User-Agent
Этот заголовок содержит информацию в нескольких частях:
-имя и версия браузера
-название и версия ОС
-язык по умолчанию

Accept-Language
Этот заголовок отображает настройки языка по умолчанию.
Если сайт имеет разные языковые версии, он может перенаправить нового пользователя на основе этих данных.

Он может содержать несколько языков, разделённых запятыми. 
Первый - это предпочтительный язык, и каждый из перечисленных языков может иметь значение «q», которое представляет собой оценку предпочтения пользователя для языка (min. 0 max. 1). 

Accept-Encoding
Большинство современных браузеров поддерживают gzip и отправляют это в header. 
Затем веб-сервер может отправить выходной HTML-код в сжатом формате. 
Это позволяет уменьшить размер до 80% для экономии пропускной способности и времени.

If-Modified-Since
Если веб-документ уже сохранен в кеше в браузере и вы посещаете его снова, 
ваш браузер может проверить, был ли документ обновлён, отправив следующее: "If-Modified-Since: ...."
Если он не изменялся с этой даты, сервер отправляет код ответа «304 Not Modified», 
а содержимое - нет, и браузер загружает содержимое из cache.

Cookie
Отправляет файлы cookie, хранящиеся в браузере для этого домена.

Referer
Данный HTTP-заголовок содержит ссылочный url.

Authorization
Когда веб-страница запрашивает авторизацию, браузер открывает окно входа в систему. 
Когда вы вводите имя пользователя и пароль в этом окне, браузер отправляет другой HTTP-запрос, но на этот раз он содержит этот заголовок.
